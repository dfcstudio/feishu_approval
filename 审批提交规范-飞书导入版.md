# 审批提交规范

> 版本：V1.0  
> 适用范围：公司内部各类审批、费用报销、采购付款、合同用印等事项  
> 归口部门：财务及相关管理部门  
> 生效日期：待定

## 1. 目的

为提高审批效率和材料质量，确保审批事项、费用明细与证明材料真实、完整、准确对应，减少附件错传、漏传、重复上传及无效凭证等情况，制定本规范。

本规范同时作为审批助手进行材料识别、完整性检查、风险提示和辅助审批的依据。

## 2. 适用范围

本规范适用于公司内部通过审批系统提交的各类申请，包括但不限于：

- 费用报销
- 差旅报销
- 采购及付款申请
- 业务招待申请
- 合同及用印申请
- 借款、还款申请
- 其他需要提交附件或证明材料的审批事项

如财务、税务、采购或其他专项制度有更严格要求，以专项制度为准。

## 3. 基本提交原则

### 3.1 事项真实

申请人应如实填写审批事由、金额、日期、供应商、费用类型、项目及其他信息，不得虚构、隐瞒或不合理拆分审批事项。

### 3.2 信息完整

审批单中的必填字段必须填写完整，不得使用“其他”“见附件”“同上”等模糊内容代替必要说明。

### 3.3 一一对应

每条报销或付款明细必须与相应凭证建立关联，确保审批人员能够判断：

- 该笔费用因何发生
- 实际支付给谁
- 发生及支付金额是多少
- 提交了哪些证明材料
- 凭证与申请事项是否一致

### 3.4 分类上传

附件应按照实际材料类型分别上传，不得将发票、支付凭证、订单截图、合同和情况说明全部归入同一类别。

### 3.5 清晰有效

附件内容应完整、清晰、可辨认、可正常打开，不得存在严重裁切、遮挡、模糊、倒置或关键信息缺失等问题。

## 4. 审批单填写要求

| 信息类别 | 填写要求 |
| --- | --- |
| 审批类型 | 选择与实际事项一致的审批类型 |
| 申请人及部门 | 按实际费用承担人或业务经办人填写 |
| 审批事由 | 说明时间、对象、事项及业务目的 |
| 费用类型 | 按实际支出性质选择，不得笼统归类 |
| 发生日期 | 与发票、订单、合同或支付日期合理对应 |
| 供应商或收款方 | 与合同、发票或付款账户信息保持一致 |
| 金额及币种 | 与明细及有效凭证金额一致 |
| 项目或成本中心 | 按费用实际归属填写 |
| 支付方式 | 如实选择个人垫付、公司支付、借款冲销等 |
| 特殊情况说明 | 对缺票、金额差异、跨期报销等情况作出说明 |

同一审批中包含多种费用类型、多个供应商或多个发生日期的，应分别填写明细，不得合并成无法核对的汇总项目。

## 5. 附件提交要求

### 5.1 附件分类

| 附件类型 | 主要内容 |
| --- | --- |
| 发票 | 增值税发票、电子发票、财政票据等合法有效票据 |
| 支付凭证 | 银行回单、银行卡交易记录、支付平台支付记录等 |
| 业务证明 | 订单、消费清单、行程单、验收单、签收单等 |
| 合同或协议 | 合同、报价单、采购订单及相关补充协议 |
| 前置审批 | 事前申请、预算审批、采购审批或招待审批等 |
| 情况说明 | 缺票、票据异常、金额差异、延期提交等书面说明 |
| 其他材料 | 与本次审批直接相关的补充证明 |

申请人不得故意选择错误的附件类别规避必交材料检查。

### 5.2 文件格式

- 发票原始文件：PDF、OFD
- 扫描件或照片：JPG、JPEG、PNG
- 表格及清单：XLS、XLSX、PDF
- 合同及说明文件：PDF、DOC、DOCX

电子发票应优先上传原始 PDF 或 OFD 文件，不应仅上传电子发票截图。

文件应能够正常打开，原则上不得上传压缩包、加密文件或需要特殊软件才能查看的文件。确需上传的，应同时提供可直接查看的版本。

### 5.3 图片质量

- 文字、金额、日期、发票号码等信息清晰可辨
- 凭证四角及边缘完整
- 图片方向正确
- 无明显反光、阴影或遮挡
- 不得涂改、伪造、拼接或隐藏关键信息
- 一张图片原则上只包含一份凭证

涉及个人敏感信息的，可遮挡与审批无关的信息，但不得遮挡姓名、交易对方、金额、时间及交易状态等核验所需内容。

### 5.4 文件命名

建议采用以下格式：

`明细序号-附件类型-供应商或事项-金额-日期`

示例：

- `01-发票-某酒店-800元-20260710.pdf`
- `01-订单-某酒店-800元-20260710.png`
- `02-支付凭证-出租车-65元-20260711.jpg`

## 6. 各类审批材料要求

### 6.1 费用报销

原则上应提交：

- 合法有效发票
- 支付凭证
- 与费用相关的订单、清单或业务证明
- 按制度要求取得的前置审批

发票、支付凭证及申请金额应能够相互印证。

### 6.2 差旅报销

根据实际费用提交：

- 出差申请或相关事前审批
- 交通票据、电子客票或行程单
- 住宿发票及住宿订单
- 支付凭证
- 其他差旅费用证明

多人出差或合并支付的，应注明同行人员和费用分摊方式。

### 6.3 采购及付款申请

原则上应提交：

- 采购申请或预算审批
- 合同、采购订单或报价材料
- 发票
- 验收单、签收单或服务完成证明
- 收款账户信息
- 按要求提供的付款进度或付款条件说明

预付款尚未取得发票的，可按制度提交合同及付款依据，但应注明后续取得发票的时间和责任人。

### 6.4 业务招待

原则上应提交：

- 事前审批
- 招待事由
- 招待对象及人数
- 消费发票
- 消费明细
- 支付凭证

不得仅以支付截图或无明细的消费记录作为完整报销依据。

### 6.5 合同及用印申请

原则上应提交：

- 完整合同或待用印文件
- 合同相对方信息
- 报价、评审或采购依据
- 法务审查意见（如适用）
- 相关业务审批记录

提交版本应为拟签署或拟用印的最终版本。

## 7. 凭证与明细关联要求

### 7.1 每条明细必须关联凭证

每条费用明细至少应关联一份核心凭证，并根据事项补充支付凭证和业务证明。

示例：

```text
明细1：住宿费800元
├─ 住宿发票
├─ 酒店订单
└─ 支付凭证

明细2：交通费260元
├─ 交通发票1：100元
├─ 交通发票2：160元
└─ 行程记录
```

### 7.2 一份凭证对应多条明细

如一份发票或付款凭证对应多条明细，应注明：

- 对应的明细序号
- 各明细分摊金额
- 分摊依据
- 合计金额

### 7.3 一条明细对应多份凭证

一条明细包含多笔实际支出的，应逐份关联凭证，确保凭证金额合计与申报金额一致。

### 7.4 附件数量异常

审批事项较少但上传附件较多时，申请人应对附件进行分类并逐份关联明细。审批助手可以发出数量异常提示，但不得仅因附件数量较多直接认定申请不合规。

存在以下情形时，应补充说明：

- 报销明细较少但凭证数量明显较多
- 单条明细关联大量票据
- 多份发票对应一笔汇总支付
- 同一附件关联多条明细
- 附件数量与申报事项明显不匹配

## 8. 发票及替代材料要求

### 8.1 应提交发票的情形

公司制度或财税规定要求取得发票的，必须提交合法有效发票。

以下材料原则上不能代替发票：

- 支付截图
- 银行交易记录
- 订单截图
- 聊天记录
- 收据或手写证明
- 商品照片
- 情况说明

上述材料可以作为辅助证明，但不能自动替代必须提交的发票。

### 8.2 无法取得发票的情形

确因客观原因无法取得发票的，申请人应：

- 选择“无票”或“发票缺失”情形
- 说明无法取得发票的具体原因
- 提交支付凭证和业务发生证明
- 提交制度要求的例外审批
- 说明相关税费或损失的承担方式

是否允许报销，由财务制度及审批权限决定。

### 8.3 发票信息要求

发票原则上应满足：

- 购买方名称及税号正确
- 发票内容与实际业务一致
- 发票金额与申请金额相符，或差异有合理说明
- 发票日期与业务发生日期合理对应
- 发票状态有效
- 不存在重复报销或重复提交
- 发票抬头、供应商与业务信息不存在明显冲突

## 9. 禁止提交的材料

不得提交下列材料作为有效审批依据：

- 与审批事项无关的文件或图片
- 无法打开、空白或严重模糊的附件
- 重复上传的同一凭证
- 经涂改、伪造或不当拼接的凭证
- 用其他申请的附件替代本次申请材料
- 仅有付款页面但无法识别收款方、金额或交易状态的截图
- 仅有商品或消费场景照片但无交易依据的图片
- 将应提交的发票替换为订单、聊天记录或情况说明
- 已作废、冲红或失效的发票，且未作出说明
- 包含恶意程序或与审批无关的大量文件

## 10. 审批助手校验规则

### 10.1 硬性拦截

出现以下情况时，原则上不允许直接提交：

- 必填字段缺失
- 明细未关联任何附件
- 制度明确要求发票但未上传发票，且未申请无票例外
- 附件无法打开或文件为空
- 申请总金额与明细合计金额不一致
- 缺少审批类型规定的核心材料
- 上传文件类型不受支持

### 10.2 风险预警

出现以下情况时，系统应提示申请人确认，必要时转人工复核：

- 申请事项较少但附件数量明显过多
- 附件与明细未明确对应
- 疑似重复发票或重复附件
- 发票金额与申报金额不一致
- 发票抬头、供应商、日期或内容异常
- 发票、支付记录与业务证明之间存在冲突
- 发票日期与业务发生日期相距较长
- 一条明细包含多份票据但未提供汇总或分摊说明
- 提交了订单或支付截图，但缺少应提交的发票
- 图片模糊、裁切、遮挡或疑似经过不当修改
- 附件内容与所选附件类别不一致

### 10.3 人工复核

以下情况不得仅由审批助手自动判断，应转人工复核：

- 无票报销或以其他材料替代发票
- 跨期、超标准或超预算申请
- 金额差异无法自动核对
- 多人、多项目或多部门费用分摊
- 异常数量的凭证集中报销
- 疑似重复报销、虚假材料或篡改凭证
- 其他制度规定的例外事项

## 11. 退回与补充材料

审批材料不符合要求时，审批人或审批助手应明确说明退回原因及需要补充的材料，不应仅填写“材料不全”“附件有误”等笼统意见。

### 11.1 缺少发票提示模板

> 本费用类型要求提交发票。当前附件识别为支付凭证或订单，不能替代发票。请补充有效发票；如确实无法取得发票，请选择无票情形，填写原因并提交例外审批材料。

### 11.2 附件过多且未关联提示模板

> 本申请包含少量报销明细和较多附件，目前无法确认各附件对应的费用。请按明细关联并分类上传；重复或无关附件请删除。

### 11.3 金额不一致提示模板

> 申请金额、费用明细与凭证金额存在差异。请核对金额；如差异合理，请填写差异原因并标明对应凭证。

## 12. 申请人提交前检查清单

- [ ] 审批类型选择正确
- [ ] 事由填写清楚
- [ ] 明细拆分合理
- [ ] 明细合计与申请总额一致
- [ ] 每条明细均已关联对应凭证
- [ ] 应提交的发票已经上传
- [ ] 发票、支付凭证和业务证明可以相互印证
- [ ] 多票据或多明细的分摊关系已经说明
- [ ] 附件清晰、完整、方向正确且能够打开
- [ ] 不存在无关、重复或错误附件
- [ ] 特殊情况已填写说明并完成例外审批

## 13. 责任要求

申请人对审批信息及材料的真实性、完整性和准确性负责。审批助手的识别、校验及风险提示仅作为辅助，不免除申请人、审批人及相关审核岗位的责任。

发现虚假申报、重复报销、伪造或篡改材料等情形的，应按照公司财务制度及相关管理规定处理。

## 14. 附则

本规范由财务及相关管理部门负责解释和维护，并根据业务流程、财税规定及审批系统能力适时更新。

各审批类型的具体材料清单、必填字段、金额阈值和校验规则，可作为本规范附件另行发布。

## 附件：审批材料快速对照表

| 审批类型 | 核心材料 | 常见辅助材料 | 典型异常 |
| --- | --- | --- | --- |
| 费用报销 | 发票、费用明细 | 支付凭证、订单 | 有支付记录但无发票 |
| 差旅报销 | 出差审批、交通或住宿票据 | 行程单、订单、支付凭证 | 日期不符、多人费用未分摊 |
| 采购付款 | 合同或订单、发票、验收材料 | 报价单、付款条件说明 | 供应商或金额不一致 |
| 业务招待 | 事前审批、发票、消费明细 | 支付凭证、对象及人数说明 | 只有支付截图、无消费明细 |
| 合同用印 | 最终合同文本 | 评审材料、法务意见 | 上传版本不是最终版本 |
