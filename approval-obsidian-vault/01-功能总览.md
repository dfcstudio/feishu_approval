# 功能总览

## 已实现功能

### 飞书审批接入

- 响应飞书 URL 校验 `challenge`。
- 校验 verification token。
- 解密带 `encrypt` 字段的 AES-256-CBC 回调。
- 解析 `PENDING` 和 `APPROVED` 审批事件。
- 获取审批实例详情、当前审批人、申请人及部门信息。
- 下载审批附件，支持飞书文件 token 和受控 HTTPS 直链。

### 审核处理

- 按可配置字段名称提取费用汇总、付款凭证、发票、申请人和办理人。
- 支持多个付款凭证和多个发票，按类别分别汇总。
- OCR 提取金额、交易号、付款时间、收款方与置信度。
- 使用 SHA-256、交易号、感知哈希和组合条件与历史记录查重。
- 生成 `LOW`、`MEDIUM`、`HIGH`、`UNKNOWN` 风险等级。
- 对同一审批进行幂等控制，避免重复处理和重复通知。

### 可靠性

- Webhook 先写入数据库任务，再由后台 Worker 处理。
- 任务使用租约避免多个 Worker 同时处理。
- 失败任务指数退避重试，重试次数可配置。
- 飞书消息通过数据库 Outbox 异步发送。
- 通知失败不会抹掉已经完成的审核结果。
- `APPROVED` 可复用 `PENDING` 阶段缓存的 OCR 结果，避免重复 OCR。

### 通知

- 发送交互式飞书卡片。
- 卡片包含审批信息、OCR 字段、汇总结果、风险原因和重复匹配摘要。
- 支持固定财务目标、申请人、当前审批人、办理人、部门财务和风控负责人。
- 提供“查看审批详情”按钮。

## 当前不做的事情

- 不自动通过或拒绝审批。
- 不回写人工确认结论。
- 没有独立的财务管理后台。
- 没有图形化风险规则编辑器。
- 本地存储 Provider 尚未替换成 S3 等对象存储。
- 感知哈希对大幅裁剪、重绘和复杂压缩的识别能力有限。
