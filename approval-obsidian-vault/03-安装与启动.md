# 安装与启动

## 前置条件

- Node.js 20 或更高版本。
- PostgreSQL。
- 已创建并发布的飞书企业自建应用。
- 如使用 AI OCR：对应模型服务的 API Key。
- 如处理 PDF：当前实现调用 macOS `/usr/bin/sips` 转换 PNG，因此部署环境需要 macOS 或替换转换逻辑。

## 首次安装

```bash
cd /Users/dfcpql/project/feishu_approval
npm install
cp .env.example .env
npx prisma generate
npx prisma migrate deploy
npm run dev
```

开发环境也可以使用：

```bash
npm run prisma:migrate
```

`prisma migrate dev` 会创建开发迁移，不应在生产部署流程中使用。

## 验证启动

```bash
curl http://localhost:7319/healthz
```

预期：

```json
{"ok":true}
```

## 常用命令

| 命令 | 用途 |
|---|---|
| `npm run dev` | 使用 `tsx watch` 启动开发服务 |
| `npm run build` | 编译 TypeScript 到 `dist` |
| `npm start` | 运行 `dist/server.js` |
| `npm test` | 运行全部 Vitest 测试 |
| `npm run test:watch` | 监听测试 |
| `npm run prisma:generate` | 生成 Prisma Client |
| `npm run prisma:migrate` | 运行开发迁移 |
| `npm run db:studio` | 打开 Prisma Studio |

## 生产启动顺序

```bash
npm ci
npx prisma generate
npx prisma migrate deploy
npm run build
NODE_ENV=production npm start
```

生产模式必须配置：

- `FEISHU_VERIFICATION_TOKEN`
- `FEISHU_ENCRYPT_KEY`

缺少任一项时，应用会在启动阶段报错退出。

## 网络入口

将公网 HTTPS 地址转发到本地服务，并在飞书设置：

```text
POST https://你的域名/webhooks/feishu/approval
```

反向代理应保留 JSON 请求体，并限制请求体大小。应用自身的 JSON 上限为 2 MB。
