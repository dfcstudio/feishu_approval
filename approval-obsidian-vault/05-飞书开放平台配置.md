# 飞书开放平台配置

## 应用设置

1. 创建企业自建应用。
2. 开启机器人能力。
3. 申请并发布所需权限。
4. 将应用安装到目标企业。
5. 配置事件订阅 URL。
6. 订阅目标审批定义的实例状态变更事件。

常见所需能力：

- 获取 tenant access token；
- 读取审批实例；
- 下载审批附件；
- 发送机器人消息；
- 读取通讯录用户基础信息，用于姓名和部门解析。

具体权限名称可能随飞书平台更新，以控制台当前说明为准。

## 事件订阅

事件 URL：

```text
https://你的域名/webhooks/feishu/approval
```

保存 URL 时，服务会返回：

```json
{"challenge":"飞书传入的 challenge"}
```

系统只处理能识别出实例编号且状态为 `PENDING` 或 `APPROVED` 的审批事件。

## 加密回调

当前代码已经支持加密回调：

- 使用 `FEISHU_ENCRYPT_KEY` 的 SHA-256 结果作为 AES-256-CBC key；
- 使用 key 的前 16 字节作为 IV；
- 解密后重新进行 payload 结构校验和 token 校验。

生产环境强制要求配置 `FEISHU_ENCRYPT_KEY`。收到加密回调但缺少密钥时返回 401。

## 审批表单字段映射

解析逻辑按控件显示名称匹配。建议确保表单中至少存在：

- 费用明细汇总或报销金额；
- 一个付款凭证或发票附件字段；
- 申请人字段；
- 如需审批后通知，增加办理人字段。

示例：

```env
APPROVAL_AMOUNT_FIELD_NAMES=费用明细汇总,报销金额,金额,实付金额
APPROVAL_ATTACHMENT_FIELD_NAMES=付款凭证,支付截图,报销凭证,附件
APPROVAL_INVOICE_FIELD_NAMES=发票,发票附件,电子发票,发票图片
APPROVAL_APPLICANT_FIELD_NAMES=申请人,报销人
APPROVAL_HANDLER_FIELD_NAMES=办理人,经办人,付款办理人
```

付款和发票字段可以出现多个；附件会按 `fileToken` 去重。发票和付款凭证分别标记为 `INVOICE` 与 `PAYMENT`。

> [!warning]
> 若同一个 file token 同时出现在付款与发票字段中，后解析的发票类别会覆盖前面的付款类别。

## 申请人姓名与部门

优先级：

1. 审批表单或实例详情中的可读姓名；
2. `APPLICANT_NAME_MAP`；
3. 飞书通讯录 API；
4. 保留原始用户 ID。

部门 ID 优先来自审批实例；缺失时尝试通过通讯录 API 查询。查询失败不会中断审核，但部门通知规则可能无法匹配。

## 审批详情链接

默认使用飞书审批 AppLink。自定义模板可包含 `{instanceCode}`：

```env
FEISHU_APPROVAL_DETAIL_URL_TEMPLATE=https://example.test/approval/{instanceCode}
```

若模板没有占位符，系统按结尾格式追加实例编号或 `instance_code` 查询参数。

## 上线校验

- URL 校验成功；
- `PENDING` 事件进入 `approval_audit_runs`；
- 能读取审批详情和附件；
- 申请人姓名和部门正确；
- 固定财务目标能收到卡片；
- `APPROVED` 后办理人只收到一次结果；
- 卡片中的审批详情链接可打开。
